# وثيقة المواصفات الفنية لتطبيق الآلة الحاسبة (Mobile-First)
**تاريخ الإصدار:** 2026-07-13
**المشروع:** app
**المنصة:** Web Mobile-First (React + Tailwind)

---

## 1) ملخص تنفيذي (Executive Summary)
تطبيق الآلة الحاسبة هو واجهة تفاعلية تعمل عبر الويب مصممة خصيصاً كـ (Mobile-First) لتقديم تجربة حوسبة سلسة وسريعة للمستخدمين على الهواتف الذكية. يتميز التطبيق بالعمل بدون اتصال بالإنترنت (Client-only)، مع ميزات متقدمة تشمل تخزين تاريخ العمليات، دوال الذاكرة (M+, M-, MR, MC)، ودعم كامل للسمتين الداكنة والفاتحة والتوطين (العربية/الอังกฤษية)، مما يجعله أداة إنتاجية يعتمد عليها للاستخدام اليومي.

---

## 2) قائمة كاملة بالميزات الوظيفية (Functional Features)

### 2.1 الميزات الأساسية (Core Features)
- **اسم الميزة:** العمليات الحسابية الأساسية
- **الوصف:** دعم الجمع، الطرح، الضرب، القسمة، النسب المئوية، وتغيير الإشارة.
- **سيناريو الاستخدام (Happy Path):** يضغط المستخدم على `1`، ثم `+`، ثم `2`، ثم `=`. النتيجة `3` تظهر فوراً.
- **سيناريو الاستخدام (Error Path):** يضغط المستخدم `1`، ثم `÷`، ثم `0`، ثم `=`. تظهر رسالة "Error" أو "خطأ".
- **معايير القبول:** الدقة الرياضية (حتى 10 خانات عشرية لمنع أخطاء النقطة العائمة العادية)، معالجة القسمة على صفر بشكل آمن.

### 2.2 الذاكرة (Memory Strip)
- **اسم الميزة:** إدارة الذاكرة (M+, M-, MR, MC)
- **الوصف:** حفظ رقم في الذاكرة للرجوع إليه لاحقاً.
- **معايير القبول:** الضغط على `M+` يضيف الرقم الحالي للذاكرة، `MR` يستدعيه للشاشة، الذاكرة تُحفظ في `localStorage`.

### 2.3 السجل (History Drawer)
- **اسم الميزة:** سجل العمليات السابقة
- **الوصف:** قائمة منبثقة (Drawer) من أسفل الشاشة تعرض آخر العمليات الحسابية وتاريخها ووقتها.
- **معايير القبول:** حفظ حتى 500 عملية، بقاء السجل بعد تحديث الصفحة، وجود زر مخصص لمسح السجل.

### 2.4 التوطين وإمكانية الوصول (Internationalization & A11y)
- **الوصف:** دعم اتجاه `RTL` للغة العربية و `LTR` للإنجليزية، مع فواصل أرقام صحيحة `Intl.NumberFormat`. دعم الشاشات القارئة (ARIA Labels) بأزرار قياس لا يقل عن 44x44 بكسل.

## 3) تصميم واجهات المستخدم (Mobile - التفصيل الممل جداً)

### 3.1 القياسات المرجعية (Breakpoints & Layout)
- **القياس الأساسي:** 375px عرض (متوافق مع iPhone X/11/12/13).
- **التصرف الشاشات الكبيرة (Desktop):** سيأخذ التطبيق `max-w-md` (حوالي 448px كحد أقصى) وسيتمركز في منتصف الشاشة مع حواف دائرية `rounded-[2.5rem]` لمحاكاة شكل الهاتف المحمول.
- **مقياس المسافات (Spacing Scale):** يتم الاعتماد على Tailwind: `p-2 (8px)`, `p-4 (16px)`, `p-8 (32px)`.
- **مساحة اللمس:** كل الأزرار في الآلة الحاسبة تحتوي على `min-h-[72px]` و `min-w-[72px]`.

### 3.2 نظام الألوان (Color System & Tokens)
| العنصر | Light Theme (HEX) | Dark Theme (HEX) | CSS Variable |
|--------|------------------|-----------------|-------------|
| Primary | `#0A84FF` | `#0A84FF` | `--color-primary` |
| Primary Hover| `#0066CC` | `#0066CC` | `--color-primary-700` |
| Background | `#F9FAFB` (gray-50)| `#09090B` (zinc-950)| `--color-bg` |
| Keys (Default)| `#F3F4F6` (gray-100)| `#18181B` (zinc-900)| `--color-key` |

### 3.3 الطباعة (Typography)
- **الخط الأساسي:** الخطوط الافتراضية للنظام (Apple System, Roboto) وأرقام بخط `font-sans` للعمليات. 
- **شاشة العرض (Display):** 
  - الإدخال الحالي: `font-size: 48px` (أو `text-5xl`).
  - الإدخال السابق (المعادلة): `font-size: 18px` مع `font-mono`.

### 3.4 مكونات الواجهة والنصوص (Microcopy)
| العنصر (ID) | الإنجليزية | العربية | الحالة |
|------------|----------|----------|--------|
| Title_Hist | History | السجل | Drawer Open |
| Empty_Hist | No calculations yet | لا توجد عمليات محفوظة | Drawer Empty |
| Clear_Hist | Clear History | مسح السجل | Drawer Action |
| Error_Msg | Error | خطأ | Display Error |

## 4) خرائط التدفق وسيناريوهات الحالة (Flows & State)

**خريطة حالة `AppProvider`:**
```typescript
interface AppState {
  theme: 'light' | 'dark';
  locale: 'ar' | 'en';
  currentInput: string;
  previousInput: string;
  operator: string | null;
  memory: number;
  history: CalcEntry[];
  isNewInput: boolean;
  error: string | null;
}
```

**تدفق ضغطة الزر (`+`):**
1. إرسال حدث `CHOOSE_OPERATOR` مع payload `+`.
2. هل يوجد `previousInput`؟ 
   - نعم: تقييم `previousInput operator currentInput` وحفظه في `previousInput`.
   - لا: نقل `currentInput` إلى `previousInput`.
3. تعيين `isNewInput = true` ليتم استبدال الرقم عند الضغط القادم.

## 5) مكونات React / API داخلية
تمت كتابة الكود لضمان الاستقلالية والأداء (أنظر ملفات المشروع المرفقة).
- `CalcKey.tsx`: يمثل زر واحد يقبل أنواع مختلفة (Operator, Function, Default).
- `CalcDisplay.tsx`: شاشة العرض مقسمة إلى سطر علوي (المعادلة) وسطر سفلي (الإدخال) مع تنسيق الأرقام بـ `Intl.NumberFormat`.
- `AppContext.tsx`: يستخدم `useReducer` لإدارة محرك الآلة الحاسبة.
**استراتيجية التقييم:** يستخدم تقييم آمن للمعادلات الثنائية للابتعاد عن مشاكل `eval()` ومشاكل `0.1 + 0.2` عبر تطبيق التقريب `toPrecision(12)`.

## 6) التخزين المحلي و History
- **التقنية:** `localStorage`
- **التخزين (Schema):** الكائن المحفوظ تحت مفتاح `calcState` يحتوي على الذاكرة، والتاريخ، والثيم، واللغة.
- **حدود التخزين:** يتم اقتطاع الـ Array برمجياً عند 500 عملية `.slice(0, 500)` لضمان عدم استهلاك الذاكرة.

## 7) الاختبارات (Testing Strategy)
- **Unit Tests (Jest):**
  - اختبار `reducer` بشكل مستقل لمعرفة نتائج العمليات.
  - مثال: إرسال `ADD_DIGIT '2'`, ثم `CHOOSE_OPERATOR '*'`, ثم `ADD_DIGIT '3'`, ثم `CALCULATE`. التحقق أن `currentInput === '6'`.
- **E2E (Playwright):**
  - محاكاة ضغطات المستخدم على واجهة الهاتف للتأكد من عدم وجود أخطاء انزياح أو Scroll غير مرغوب.

## 11) الأمن والخصوصية (Security & Privacy)
- التطبيق يعمل محلياً 100% (Client-only). لا يتم إرسال أي بيانات حسابية لأي خادم خارجي.
- الـ `localStorage` يستخدم لتخزين العمليات الحسابية فقط ولا يخزن أي بيانات حساسة PII.
- يُنصح بتفعيل `Content-Security-Policy` لمنع الحقن الخبيث للسكربتات.
